Terminologie

Repository

  • Verzeichnis fuer Datenspeicherung
  • Typischerweise auf Servern (oder Peers)
  • Versionsverwaltung
  • Protokolle von Veraenderungen
  • Konfliktaufloesung
  • Metadaten fuer Paketmanager

Clone

  • Duplikat des Repositories
  • Backup-Aspekt
  • Offline-Verfuegbarkeit
  • Dezentrale Nutzung

Beispiel

mkdir ./lycheeJS;
git clone https://github.com/Artificial-Engineering/lycheeJS.git;

Commit

  • Veraenderung einer Datei
  • Veraenderung zu einem Zeitpunkt
  • Entfernen (-) oder Hinzufuegen (+)
  • Zeilenbasiert bei Textuellem Inhalt
  • Dateibasiert bei Binaerem Inhalt

Beispiel

mkdir ./lycheeJS;
git clone https://github.com/Artificial-Engineering/lycheeJS.git;

echo "Awesome Content" > newfile.txt;
git add ./newfile.txt;
git commit -m "Awesome new content";

Revision

  • Synonym fuer Version (Hash)
  • Entwicklungsstadium der Software
  • Einzelner Commit

Tag

  • Entwicklungszeitpunkt
  • Version der Software
  • Serie an Commits

Beispiel

mkdir ./lycheeJS;
git clone https://github.com/Artificial-Engineering/lycheeJS.git;

echo "Awesome Content" > newfile.txt;
git add ./newfile.txt;
git commit -m "Awesome new content";

git tag v1.0 -m "Software is stable nao.";
git push origin --tags;

Branch

  • Abzweigung von master-Branch
  • Variante der Software
  • Feature (Serie an Commits)

Beispiel


git clone https://github.com/Artificial-Engineering/lycheeJS.git;

git checkout development;
git checkout -b my-feature;

echo "bla" > datei.txt;
git commit -m ":sparkles: Started work on my-feature";

Fork

  • Abzweigung des Projekts
  • Fork ist neue Software
  • Upstream ist alte Software
  • Bsp. LibreOffice ist Fork von OpenOffice

Triangular Workflow

Uebersicht

1. Pull

  • Pull von Upstream Repository
  • Upstream Repository ist behind

2. Commit

  • Lokaler Merge von Commits
  • Lokale Feature Branches
  • Lokale Commits

3. Push

  • Push zu eigenem Fork (Origin)
  • Origin Repository ist ahead
  • Keine Merge Conflicts

4. Pull Request

  • Fork Maintainer initiiert Pull Request
  • Upstream Maintainer merged Pull Request
  • Keine Merge Conflicts
  • Alle sind gluecklick
  • Suck it, SVN und CVS

Wikis

Projekt Setup

Aufgabe

  1. Installiere git
  2. Forke die Boilerplate [1]
  3. Clone deinen Fork lokal
  4. Fuege den Inhalt der HTML5 Datei hinzu
  5. Erstelle einen Feature Branch
  6. Habe Commits von allen Teammitgliedern
  7. Erstelle einen Pull Request
